## 1. Что делаем

Админ-**тумблер**, открывающий/закрывающий приём сертификационных завтраков **по когорте**. По умолчанию **ЗАКРЫТ**.

**Зачем:** фича выкатывается на прод готовой, но закрытой. Ольга открывает приём **сама** в нужный день (≈через 2 недели, когда ведущие готовы сдавать), без обращения к разработке. **Переиспользуется на будущие потоки** — у каждой когорты свой переключатель (принцип «продумать один раз»).

**Поведение:**
- **Закрыто (default):** на «Моей странице» вместо бланка — спокойное состояние «**Приём сертификационных завтраков откроется позже**». Страница правил «Сертификация» видна всегда (можно читать заранее). Рабочий бланк реальные ведущие НЕ видят. Менторский блок тоже закрыт.
- **Открыто:** бланк самооценки активен (менти заполняет), менторская оценка активна, после двух submit — сравнение (как уже построено).
- **Admin** (Ольга/Настя/Ира): видит бланк/сравнение **всегда**, независимо от флага — для превью и проверки перед открытием.

## 2. Storage — миграция phase42

```sql
ALTER TABLE pvl_cohorts
  ADD COLUMN certification_open boolean NOT NULL DEFAULT false;
```
- Per-cohort, default `false` → все существующие когорты закрыты.
- Данные не мигрируем (`pvl_cohorts` существует).
- В конце миграции — `SELECT public.ensure_garden_grants();`
- **Сначала DRYRUN** (`BEGIN … ROLLBACK`, `\d pvl_cohorts` после), apply только после 🟢.

## 3. RLS / доступ

- **SELECT `certification_open`** — читают члены когорты (студент/ментор должны знать, открыт ли приём). Проверь текущие политики `pvl_cohorts`: если `authenticated` уже читает cohorts — достаточно; иначе добавь SELECT для своей когорты / `has_platform_access(auth.uid())`.
- **UPDATE `certification_open`** — ТОЛЬКО `is_admin()`. Проверь/добавь UPDATE-политику `pvl_cohorts` для admin. Студент/ментор менять НЕ могут.

## 4. API — `services/pvlPostgrestApi.js`

- `getCohortCertificationOpen(cohortId)` → `GET /pvl_cohorts?id=eq.{id}&select=certification_open`
- `setCohortCertificationOpen(cohortId, open)` → `PATCH /pvl_cohorts?id=eq.{id}` body `{ certification_open: open }` (admin)
- Если загрузка когорты уже где-то есть — встрой чтение флага туда, не плоди лишний запрос.

## 5. Frontend gating

- **`PvlCertificationBlock`** (на peer / «Моя страница»): прочитать `certification_open` когорты студента.
  - `open=false` И зритель не admin → спокойное **locked-состояние** «Приём сертификационных завтраков откроется позже» (дружелюбный текст, НЕ старая битая заглушка). Wizard не монтировать.
  - `open=true` → текущее поведение (wizard self/mentor + сравнение).
  - **admin** → всегда активно (превью), независимо от флага.
- **Кнопка-мост** «Открыть бланк на моей странице →» (страница правил): при закрытом приёме ведёт на «Мою страницу» → там locked-состояние. Без поломок. (Опционально: подпись «приём пока закрыт».)
- **Менторский блок** — гейтить тем же флагом когорты студента.
