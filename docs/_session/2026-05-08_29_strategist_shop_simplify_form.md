# Магазин — убрать мёртвые поля promo_code/link_url из формы

**Адресат:** VS Code Claude Code (executor).
**Автор:** стратег (claude.ai).
**Создано:** 2026-05-08.

## Контекст

После последних фиксов магазина (`f65c7b4`, `3522581`):
- chip промокода удалён с витрины.
- логика кнопок: `download_url` → `contact` (без `link_url`).
- описание поддерживает auto-link и multi-line.

→ Поля `promo_code` и `link_url` в админ-форме теперь **мёртвый
код** (заполняются Ольгой, но нигде не отображаются). Сбивают
логику: Ольга хочет создать карточку «Промокоды для ведущих» с
многострочным описанием, но форма заставляет заполнить
single-промокод и single-ссылку.

Решение — убрать секцию «Для товаров со скидкой по промокоду»
из формы. Колонки в БД **оставить** как есть (legacy-данные не
трогаем).

## Что сделать

В `views/AdminPanel.jsx`, компонент ShopAdmin, форма
редактирования товара:

1. **Удалить блок секции «Для товаров со скидкой по промокоду»**
   с полями:
   - `promo_code` (Промокод)
   - `link_url` (Ссылка перехода)

   Включая label-обёртки и саму секцию-контейнер с заголовком.

2. **Удалить поле `contact_whatsapp` (WhatsApp)** из формы.
   Ольга им не пользуется. Telegram остаётся как единственный
   контакт для кнопки «Связаться».

3. **НЕ удалять** все эти поля из:
   - state `form` (оставить, но они никогда не редактируются)
   - `SHOP_EMPTY_FORM` (можно оставить пустыми)
   - `payload` при сохранении (можно оставить как есть, чтобы
     не нарушить существующих записей; новые записи получат `null`/`''`)
   - `openEdit` инициализации (оставить для обратной совместимости)

   Это позволит существующим записям (LOVELIGA, LIGANOTEBOOKS,
   возможно записи с whatsapp) спокойно жить — их `promo_code`/
   `link_url`/`contact_whatsapp` останутся в БД, просто Ольга
   их не будет видеть/редактировать через UI. Если в будущем
   понадобится — вернём поля в форму одной строкой.

3. **Витрина** (`views/MarketView.jsx`) — НЕ трогаем. Уже не
   использует `link_url` после предыдущих фиксов. Логика кнопок
   и так download → contact.

## Что НЕ делать

- **НЕ DROP колонок в БД** — миграция не нужна.
- **НЕ удалять старые записи LOVELIGA / LIGANOTEBOOKS** — Ольга
  сама удалит их через UI после создания новой карточки.
- **НЕ менять поведение существующих записей** — если в БД у
  товара заполнено `promo_code` / `link_url` / `contact_whatsapp`,
  оно просто не отобразится на витрине (как и раньше после
  chip-фикса).
- **Витрина MarketView** — НЕ трогаем, она и так корректно
  работает с любым из contact-полей. Просто Ольга не будет
  заполнять whatsapp, кнопка «Связаться» всегда через telegram.
