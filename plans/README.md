# plans/ — технические планы реализации

Здесь живёт информация **как** мы делаем проект. Один файл = один план = одна функция.

## Правила

- **Один план — одна функция.** Если план уже есть — работаем с ним, не создаём новый.
- **Имя файла:** `YYYY-MM-DD-название-функции.md` (например, `2026-04-14-авторизация.md`).
- **Структура плана:** делится на фазы. Каждая фаза имеет статус `[ ]` (не сделано) или `[x]` (готово).
- **Итоговый блок** в конце каждого плана: реализован целиком или нет, что осталось.

## Обязанность агента

Любой агент, работающий с планом, обязан его актуализировать после сессии:
отмечать выполненные фазы, дописывать что было сделано и что осталось.
