2. **Доступность в минусе.** `ModalShell` без focus-trap/Esc/aria-modal; `prefers-reduced-motion` — **0 файлов**; контраст muted (#7A6758/#9B8B80 на #FAF6F2 ≈2.5:1, `text-slate-400` на светлом) ниже WCAG AA; тач-таргеты иконок <44px; детектор: 8× gray-on-color (MeetingCard, RichEditor). Аудитория — женщины 35+, преимущественно с телефона.

# Дорожная карта (выполнять по фазам, каждая = отдельная codeexec-сессия с diff-on-review)

Принцип порядка: дёшево+безопасно+высокий импакт → структурный корень → вкус. Каждая фаза самостоятельно verifiable.

## Фаза A — Токены-фундамент (zero-risk, открывает всё остальное)
**Что:** вынести тёплую коричневую палитру в именованные CSS-токены в `index.css @theme` (значения = текущие хексы 1:1, визуально ничего не меняется). Определить токены кривых: `--ease-out: cubic-bezier(0.23,1,0.32,1)`, `--ease-drawer: cubic-bezier(0.32,0.72,0,1)` (уже встречается в коде).
**Зачем первым:** это keystone для Фазы D, но сам по себе безрисковый (1:1 маппинг).
**Acceptance:** новые токены определены; сборка зелёная; визуальный diff = 0 (хексы те же).

## Фаза B — Доступность, быстрые победы (дёшево, бьёт по людям, низкий риск)
**Шаги (каждый — отдельный коммит):**
1. Глобальный `@media (prefers-reduced-motion: reduce)`: флип-карты/pulse/parallax → статика, opacity-переходы оставить.
2. `ModalShell` (`components/ModalShell.jsx`): `aria-modal`, закрытие по Esc, focus-trap (фокус не уходит за модалку, возврат на триггер при закрытии).
3. Контраст: muted-токены (#7A6758/#9B8B80; `text-slate-400` на светлых фонах) поднять до AA (≥4.5:1) — на тон темнее.
4. Тач-таргеты: иконочные кнопки <44px (`PvlMenteeCardView:387,466-580`, `AdminPvlProgress:232`, `PracticesView:432`, `TreasuryView:347`) → min 44×44 хит-зона.
**Acceptance:** axe/ручная проверка модалки с клавиатуры; контраст-чек пар; reduced-motion включён в системе → движения нет.

## Фаза C — Полировка ощущения (дёшево, «дороже» сразу) — emil-слой
1. `.btn-*`/`.input-field`: `transition-all duration-300` → точные свойства (`transform 150ms, background-color/border-color 200ms`) + `--ease-out`. (95 `transition-all` по проекту — пройтись по горячим.)
2. Отклик кнопок 150ms (press `scale-[0.98]` уже есть — сохранить); фокус-переходы на формы ПВЛ (`PvlTrainingFeedbackForm:97-114` и др. — сейчас focus-стилей нет).
3. `MapView` флип `group-hover:[rotateY(180deg)]` → обернуть в `@media (hover:hover) and (pointer:fine)` + tap на тач + reduced-motion фолбэк.
**Acceptance:** кнопки/поля отвечают <200ms с сильным ease-out; флип работает на тач; reduced-motion гасит флип.

## Фаза D — КОРЕНЬ: единая палитра (большая, инкрементально) — P0
1. Мигрировать хардкод-хексы кластера ПВЛ → токены Фазы A (`PvlStudentCabinetView`, `PvlMenteeCardView`, `PvlTaskDetailView`, формы, `PvlPrototypeApp.jsx`).
2. Мигрировать кластер менторских/админ-смешанных (`MentorDashboardView` и др.).
3. Свести два беж-набора (main `slate` vs ПВЛ-коричневый) к ОДНОЙ тёплой шкале; зелёный — акцент поверх нейтралей.
4. Дедуп `statusTone()` (сейчас в 3 местах: `PvlTaskDetailView`, `PvlMenteeCardView`, `PvlPrototypeApp`) → один shared util.
**Acceptance:** ноль (или near-zero) хардкод-хексов в кластерах; смена бренд-цвета = правка токенов в одном месте; Сад и ПВЛ читаются одним продуктом.

## Фаза E — Вкус и состояния (средняя)
1. Рационировать eyebrow'ы (170 `uppercase tracking-*`): оставить семантические (pill/статус), убрать декоративные над каждым блоком.
2. Ленивые равные 3-колоночные сетки (37× `grid-cols-3`) — где содержание просит — заменить на контент-управляемую раскладку/волосяные линии.
3. Стекло-по-умолчанию (11× `backdrop-blur`) → бумажная текстура + `divide-y`/`border-t` там, где стекло не заслужено; оставить glass как редкий акцент.
4. Общий `<EmptyState>`/`<ErrorState>` компонент (сейчас каждый экран лепит `text-slate-400`); placeholder-цвета и inline-валидация в формах.
**Acceptance:** eyebrow-каунт ≤ разумного; есть переиспользуемые empty/error; формы с focus+placeholder+валидацией.
