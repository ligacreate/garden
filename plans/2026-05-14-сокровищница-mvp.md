# FEAT-019 MVP — Сокровищница (минимум для студентов ПВЛ)

**Создано:** 2026-05-14
**Автор задачи:** Ольга Скребейко
**Источник ТЗ:** [`docs/_session/2026-05-07_10_idea_treasury_marketplace.md`](../docs/_session/2026-05-07_10_idea_treasury_marketplace.md)
**Связано с BACKLOG:** FEAT-019 (этот план — урезанный MVP, полная фича остаётся в P2-P3)

## Зачем MVP

Студенты курса ПВЛ должны иметь доступ к практикам Лиги (которые
сегодня лежат в личных коллекциях кураторов и недоступны другим).
Полный объём FEAT-019 (UGC-публикация, модерационная очередь, семена
за публикацию, маркетплейс, полнотекстовый поиск) — большой и
рискованный (8-11 сессий). MVP даёт **место** и **поток**:

- админ помечает практики флагом → они появляются в Сокровищнице;
- ведущая видит, фильтрует, забирает форком к себе.

Всё остальное (UGC, модерация, семена, классификаторы, маркетплейс)
— фаза 2, после ПВЛ-сессии.

## Архитектурные решения (зафиксированы 2026-05-14)

1. **Сокровищница = режим публикации существующей `practices`**, не
   отдельная таблица. Флаг `is_published` + `published_at`.
   Полиморфность для сценариев — на будущее, в MVP не закладываем.
2. **Форк = независимая копия** записи. Связь сохраняется через
   `forked_from` (FK) и `forked_from_author_name` (кэш имени на
   момент форка — переживает удаление автора и переименование).
   Никаких «обновлений оригинала с пушем в форки» — это фаза 2.
3. **Публикация — только админ в MVP.** RLS-политика
   `Practices_Update_Admin` позволит админу менять `is_published`
   на чужих практиках. UGC-публикация (с согласия автора) —
   фаза 2.

## MVP scope

### ВКЛЮЧЕНО

- Переименование «База практик» → «Мои практики» (с подзаголовком).
- Новый пункт меню «Сокровищница» — виден всем авторизованным.
- БД: `is_published`, `published_at`, `forked_from`,
  `forked_from_author_name` + RLS публичного чтения + RLS админ-UPDATE.
- Админ-кнопка «Опубликовать в Сокровищнице» / «Снять с публикации»
  на любой практике.
- Кнопка «Добавить в мою коллекцию» в Сокровищнице → форк.
- Атрибуция автора на карточке Сокровищницы и на форкнутой копии.
- Два фильтра: тема (`type`) + длительность (`duration_minutes`).

### НЕ ВКЛЮЧЕНО (фаза 2)

- UGC-публикация ведущими + модерационная очередь.
- Семена за публикацию.
- Маркетплейс (`is_for_sale`, `price`, `contact_method`,
  `purchase_log`, «Мои продажи»).
- Полнотекстовый поиск + `forks_count` + сортировка по популярности.
- Расширенные классификаторы (`practice_type`, `intensity`,
  `usage_context`, `format`, `audience_readiness`, `materials`,
  `related_practices`, `tags` как массив).
- Индикатор «обновлено автором», merge изменений с оригинала.
- Сокровищница для сценариев.

### Решение по существующим полям

- `practice_type` (enum из ТЗ) — **не вводим**. Используем
  существующее `type` (text). Подтверждено Ольгой 2026-05-14.
- Дубль `time` (text) и `duration_minutes` (integer) — **оставляем
  как есть**. Это известный технический долг (BACKLOG?), вычистим
  отдельной задачей. Парсинг `parseDurationMinutes` уже умеет:
  duration_minutes приоритетнее, time как fallback.
- Колонка `icon` (text, default '📝') — уже в БД, ничего не делаем.

### RLS сейчас

- `Practices_View_Own`  — `auth.uid() = user_id` (SELECT)
- `Practices_Insert_Own` — INSERT only own
- `Practices_Update_Own` — UPDATE only own
- `Practices_Delete_Own` — DELETE only own

Нет публичного чтения, нет админских политик.

### Allowlist в `dataService.js` (микрофикс ниже)

`_sanitizeFields()` — XSS-санитайзер, **не** allowlist на дроп.
Поля кроме `id` проходят passthrough'ем. Фактически `icon` и
`duration_minutes` доходят до PostgREST. Но контракт нечитаемый
(выглядит как allowlist) — добавляем `icon` в `plain` для ясности
и санитизации.

### Фаза 1. Миграция БД (`phase28_treasury_mvp.sql`)

Один файл, одна транзакция. SELECT `public.ensure_garden_grants()`
до COMMIT (RUNBOOK 1.3). Apply вручную по команде Ольги.

**Колонки:**
- `is_published boolean NOT NULL DEFAULT false`
- `published_at timestamptz`
- `forked_from bigint REFERENCES practices(id) ON DELETE SET NULL`
- `forked_from_author_name text`

**Индексы (partial):**
- `idx_practices_is_published WHERE is_published = true` — для
  выборки Сокровищницы.
- `idx_practices_forked_from WHERE forked_from IS NOT NULL` — для
  будущего счётчика форков.

**RLS:**
- Новая политика `Practices_View_Published` — все `authenticated`
  видят SELECT записи `WHERE is_published = true`. Существующая
  `Practices_View_Own` остаётся; PostgREST применит OR.
- Новая политика `Practices_Update_Admin` — `public.is_admin()`
  может UPDATE любую (для публикации/снятия). USING + WITH CHECK.

**NB-1: защита от подмены `is_published` пользователем.**
Сейчас `Practices_Update_Own` без column-level фильтра позволит
ведущей выставить себе `is_published = true` через DevTools. Это
обходит «публикация только админом».

Варианты:
- **A.** Триггер `BEFORE UPDATE` — если `NEW.is_published <>
  OLD.is_published` и не админ → RAISE. Чисто, нельзя обойти.
- **B.** Column-level GRANT — REVOKE UPDATE(is_published, published_at)
  от `authenticated`, GRANT отдельно для админа. Тоньше, но
  Timeweb-grant-wipeout-сценарий усложняет.

**Решение MVP:** вариант A (триггер). Закладываем в phase28.

**Файл миграции:** [`migrations/2026-05-14_phase28_treasury_mvp.sql`](../migrations/2026-05-14_phase28_treasury_mvp.sql)

### Фаза 3. Новый раздел «Сокровищница»

**Новый файл:** `views/TreasuryView.jsx`
- Загружает `is_published = true` практики через
  `dataService.getTreasuryPractices(filters)`.
- Embed автора: PostgREST select `*, profiles!practices_user_id_fkey(name)`.
- Карточка как в PracticesView (read-only), показывает имя автора.
- Кнопка «Добавить в мою коллекцию» → форк.

**Меню:** новый пункт `view='treasury'`, лейбл "Сокровищница",
иконка `Gem` или `Sparkles` (lucide). Видим всем `authenticated`.

**Новые методы в `dataService.js`:**
- `getTreasuryPractices({ type, minDuration, maxDuration })` —
  PostgREST GET `practices?is_published=eq.true&select=*,author:profiles!practices_user_id_fkey(name)`
  + фильтры.
- `forkPractice(originalPracticeId)` — клиентский путь для MVP:
  1. GET оригинала с автором.
  2. POST новой записи: title, description, short_goal,
     instruction_short, instruction_full, reflection_questions,
     type, time, duration_minutes, icon → копируются. user_id =
     current user. forked_from = original.id. forked_from_author_name
     = original.author.name. is_published = false. published_at = null.
  3. Возврат сохранённой копии в state «Мои практики».

  **NB-2:** атомарность форка не гарантирована (2 HTTP round-trip).
  Race: автор удалит оригинал между шагами → копия будет с
  forked_from = NULL после ON DELETE SET NULL. Для MVP приемлемо.
  В фазе 2 — переводим на RPC `fork_practice(p_id bigint)` с
  SECURITY DEFINER и WHERE is_published=true.

## NB на будущее (не блокирует MVP)

- **NB-1 (триггер защиты `is_published`)** — закрыто в phase28 (`trg_protect_practice_publish_flag`).

Фаза 2 — RPC `fork_practice(p_id bigint)` SECURITY DEFINER.

## Smoke-инструкция для проверки на проде

После деплоя фронта — пройти как админ и как обычная ведущая:

1. **Админ:** открыть «Админка → Контент → Практики». В таблице видны ВСЕ практики системы. Выбрать 1-2 и нажать «Опубликовать» — статус должен смениться на «В Сокровищнице» (зелёный pill).
2. **Любая ведущая (можно та же админская сессия):** открыть «Сокровищница» в сайдбаре (иконка Gem). Опубликованные практики должны появиться с атрибуцией автора.
3. Проверить фильтр по теме (pill) и по длительности (select).
4. Открыть карточку → нажать «Добавить в мою коллекцию». Перейти в «Мои практики» — копия должна быть на месте, без `is_published`. На карточке Сокровищницы кнопка должна стать «В коллекции» (disabled).
5. Админ снимает с публикации — карточка пропадает из Сокровищницы у ведущей при reload.
6. **Защита от подмены `is_published`:** в DevTools открыть Network, попытаться PATCH `practices?id=eq.<своя_практика>` body `{"is_published":true}` от обычной ведущей. Должно прийти 400/403 (триггер `trg_protect_practice_publish_flag`).

## Архитектурные решения phase28b

1. **Колонка `seeds_awarded boolean NOT NULL DEFAULT false`** в practices.
   Бэкфилл `true` для уже-опубликованных на момент apply (на проде 0,
   но защита). Гарантия «один раз»: после первого начисления флаг
   остаётся true даже после снятия и переопубликации.
2. **Триггер `protect_practice_publish_flag` ослаблен** — владелец
   может toggle `is_published` на СВОЕЙ практике, админ — на любой.
   `auth.uid() IS DISTINCT FROM OLD.user_id AND NOT public.is_admin()`
   → 42501.
3. **Триггер `award_seeds_on_first_publish`** — `AFTER INSERT OR
   UPDATE OF is_published`. Условие: `NEW.is_published=true AND
   COALESCE(NEW.seeds_awarded,false)=false AND user_id IS NOT NULL`.
   Действия: `UPDATE seeds_awarded:=true; PERFORM
   increment_user_seeds(ARRAY[user_id], 40)`. UPDATE не вызывает
   рекурсию — триггер на `OF is_published`, не на seeds_awarded.
4. **Семена при «создал и сразу опубликовал»: +80 + +40 = +120.**
   Триггер срабатывает и на INSERT с is_published=true. Решение
   Ольги (см. вопрос 2026-05-14).
5. **Админ «+ Добавить» в Сокровищнице** создаёт запись с
   `user_id=admin.id` — она ОДНОВРЕМЕННО попадает в его «Мои
   практики» и в Сокровищницу (одна запись, разные view'ы).
   Сокровищница = `is_published=true`, не отдельная сущность.
6. **Поле `time` в БД пока не дроп** — отдельная миграция после
   уверенности, что нигде не читается. В UI формы поле убрано;
   парсер `parseDurationMinutes` уже умеет приоритизировать integer.
