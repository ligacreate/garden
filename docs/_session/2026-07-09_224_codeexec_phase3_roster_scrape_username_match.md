  сохраняет по каждому участнику `user_id` **и** `username` (старый `match_to_garden.py` id выбрасывал
  и матчил по имени — тут этого нет).

  сторон (снять `https://t.me/`, `@`, lowercase). Никакого fuzzy по имени.
- Сырой ростер → `roster_phase3.json` (в `.gitignore`, не коммитим).

## Бартер (`auto_pause_exempt`) — предметно

```

всего профилей           : 60
```

**Бартер/льготники НЕ оцифрованы (0 записей) → это блокер для КИКА.** Пока exempt пуст, любой
авто-кик по «нет оплаты» слепой: выкосит бартерных/команду наравне с отвалившимися. B1-`paused_manual`
(7 профилей) сейчас держатся ручной паузой, а не флагом exempt — хрупко.

## Выводы для гейта

1. **Числовое покрытие поднимается 8 → 21** одним точным username-бэкфиллом (13 профилей, 10 из них
   оплачены). Fuzzy по имени не нужен и не использовался.
2. **Оставшиеся 20 NULL — это НЕ «дыра для кика»:** 10 из них законно вне чата (expired/manual),
   4 без username в профиле, 6 активных-но-отсутствующих (аномалии на разбор). То есть реальный
   «actionable NULL среди тех, кто в чате» невелик.
3. **10 участников чата без профиля** — параллельная задача (recon 215: «завести профиль»), их
   enforcement не покроет, пока нет `profiles.id`.
4. **Кик заблокирован до оцифровки бартера** (`auto_pause_exempt=0`).
5. Идеальный источник числовых id going-forward — сам бот-админ (`chat_member`/join-request updates),
   но и разовый username-бэкфилл уже закрывает ядро оплаченных.
