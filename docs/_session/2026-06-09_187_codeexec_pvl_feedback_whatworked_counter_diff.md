# codeexec — UX: счётчик/required на поле «Что сработало» (PvlTrainingFeedbackForm)

## Задача (от стратега)
Поле `whatWorked` — единственный gate кнопки (нужно ≥50 симв.). Участница должна
сразу видеть, ПОЧЕМУ кнопка неактивна:
- required + hint «Минимум 50 символов — пара предложений»
- живой счётчик `length/50` под полем: серый <50, зелёный ≥50

Остальные поля и логику не трогать. Билд → commit + push. Без smoke. dist/ не коммитить.

## Состояние на входе
Файл уже частично содержал правку прошлой сессии: `required`, hint и счётчик
`{...}/{MIN_WHAT_WORKED}` уже были. Не хватало двух деталей по ТЗ:
1. Счётчик всегда серый — нет зелёного при ≥50.
2. Формулировка hint отличалась: «…символов. Два-три конкретных момента.»

## Что сделано (2 точечные правки, только `whatWorked`)
1. **hint** → точная формулировка ТЗ: `Минимум ${MIN_WHAT_WORKED} символов — пара предложений.`
2. **Счётчик — условный цвет**: переиспользую существующий `valid`
   (= `whatWorked.trim().length >= MIN_WHAT_WORKED`, тот же gate кнопки) —
   зелёный (`text-green-600`) ровно когда кнопка разблокируется, иначе серый.

```diff
-                <Field label="Что в этой встрече сработало" required hint={`Минимум ${MIN_WHAT_WORKED} символов. Два-три конкретных момента.`}>
+                <Field label="Что в этой встрече сработало" required hint={`Минимум ${MIN_WHAT_WORKED} символов — пара предложений.`}>
...
-                    <div className="text-[11px] text-[#7A6758] mt-1">{whatWorked.trim().length}/{MIN_WHAT_WORKED}</div>
+                    <div className={`text-[11px] mt-1 ${valid ? 'text-green-600' : 'text-[#7A6758]'}`}>{whatWorked.trim().length}/{MIN_WHAT_WORKED}</div>
```

- `required` и звёздочка `*` — уже на месте (компонент `Field`, проп `required`).
- Счётчик и gate используют один источник правды (`.trim().length`) — счётчик
  не соврёт относительно реального состояния кнопки.
