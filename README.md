# Сад ведущих

Закрытая платформа-сообщество для ведущих письменных практик. По подписке.

## Зачем это

Ведущая прошла обучение и провела первые встречи — и осталась одна.
Нет среды, где можно расти дальше, видеть других ведущих, делиться
сценариями и материалами, держать практику живой. «Сад» закрывает эту
дыру: даёт инструменты для работы (расписание, сценарии, библиотека) и
сообщество, в котором эта работа имеет смысл.

Основная воронка: курс ПВЛ → подписка на платформу.

## Что умеет

- **Профиль ведущей.** Имя, город, навыки, оффер, уникальные
  способности, аватар, друидское дерево по дате рождения. Карта
  ведущих, публичная страница ведущей с подписью и отзывами.
- **Встречи и публичное расписание.** CRUD встречи (дата, время, TZ,
  город/онлайн, формат, цена, обложка). Чек-бокс «публичная» → встреча
  попадает в общий календарь городов. Есть публичный JSON-фид
  `/api/v1/upcoming.json` для внешних пайплайнов (Telegram-карточки).
- **Библиотека и практики.** Курсовые материалы по категориям с
  прогрессом прохождения; отдельный раздел практик с импортом из
  Notion-CSV.
- **Сценарии (Builder).** Конструктор тайм-лайна встречи: блоки,
  drag-reorder, экспорт в PDF, публичные/приватные.
- **Чат и push.** Realtime-сообщения сообщества (текст + картинки);
  web-push на десктоп и iOS PWA.
- **Подписка.** Биллинг через Prodamus: webhook → доступ
  открыт/закрыт/просрочен. Глобальный gate в RLS — без активной
  подписки большая часть данных недоступна.
- **ПВЛ — курс-в-курсе.** Программа обучения ведущих: 20 недель, ДЗ,
  контрольные точки, сертификация. Свои роли (студент / ментор / админ
  ПВЛ) и свой UI. Сейчас на mock-API в памяти, БД-схема готова, но не
  подключена.
- **Админка.** CRUD пользователей, смена ролей, статистика встреч по
  периодам/городам/ведущим, импорт знаний и практик.

## Роли

Путь участника: `Абитуриент` (applicant) → `Стажёр` (intern) → `Ведущая`
(leader). Служебные роли назначает админ: `Ментор` / `Куратор` /
`Администратор`. Иерархия и права — в [utils/roles.js](utils/roles.js).

В ПВЛ — отдельная ось ролей (студент / ментор / админ курса), не
смешивается с садовыми ролями.

## Стек

React 19 + Vite + Tailwind 4 на фронте. PostgreSQL + PostgREST как API,
плюс отдельные node-сервисы:
[push-server](push-server/) (web-push, Prodamus webhook, публичный
upcoming-фид) и auth-сервис (вне репо). Аутентификация — JWT в
localStorage. Storage — Supabase Storage (S3-совместимый). Деплой
фронта — GitHub Actions → FTP.

## Запустить локально

```bash
npm install
cp .env.example .env   # подставить VITE_POSTGREST_URL, VITE_AUTH_URL и т.п.
npm run dev            # http://localhost:5173
```

Push-сервер — отдельный пакет:

```bash
cd push-server
npm install
cp .env.example .env   # DATABASE_URL обязателен
npm run start          # http://localhost:8787
npm run test           # node --test, без БД
```

Сборка прода: `npm run build` → артефакт в `dist/`.

## Ориентация в репо

- [CLAUDE.md](CLAUDE.md) — карта репозитория и правила работы для AI-агентов и
  людей: где искать код, как заводить план, как фиксить баги.
- [plans/](plans/) — технические планы по функциям, **актуальный
  источник «как»**.
- [docs/lessons/](docs/lessons/) — что мы выучили на багах. Растёт по
  одному уроку на инцидент.
- [docs/snapshots/](docs/snapshots/) — замороженные обзоры
  (PRD, FEATURES, PROJECT_PASSPORT и т.п.). Снимки на дату создания, не
  источник правды по текущему коду.
- [docs/journal/](docs/journal/) — датированные артефакты сессий
  (EXEC_*, HANDOVER_*, REPORT_*, INCIDENT_*, …). Хронология, не справочник.
- В корне [docs/](docs/) — живые справочники: runbook, дизайн-система,
  feedback, утилитные SQL-скрипты.
- `.business/` (не в git) — продуктовый и бизнес-контекст: аудитория,
  тарифы, цели, экономика.

## Принципы продукта

- Сообщество важнее продукта. Живые связи между ведущими важнее метрик.
- Личный рост ведущей — жемчужина платформы, не просто библиотека.
- Прямых конкурентов-платформ в нише нет — стандарт задаём мы.
