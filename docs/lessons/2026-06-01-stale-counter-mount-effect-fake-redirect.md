# Ложный «редирект» из stale монотонного счётчика + level-проверка в mount-эффекте на ремаунте

**Дата инцидента:** 2026-06-01.
**Связанные сессии:** [_178 recon (где живёт соц-психология)](../_session/2026-06-01_178_codeexec_recon_elena_status_and_socpsy.md), [_179 recon + предлагаемый фикс навигации](../_session/2026-06-01_179_codeexec_recon_mentor_library_nav.md).
**Фикс:** commit `236e7c2` (`views/UserApp.jsx`).

## Симптом

Ментор в Garden-меню жмёт «Библиотека» → весь левый набор меню переключается с Garden (Дашборд/Встречи/Сад ведущих/Сценарии…) на **PVL-набор ментора**, открывается PVL «Библиотека курса». Garden `CourseLibraryView` (где курс «Социальная психология») **не открывается**. Через «Вернуться в сад» Garden-меню возвращается, но **повторный** клик «Библиотека» снова кидает в ПВЛ. SPA: **URL всё время `/`, не меняется**. Первый-первый клик «Библиотека» (до любого входа в ПВЛ) при этом отрабатывает корректно — баг проявляется только ПОСЛЕ первого входа в ПВЛ через «Учительская»/«ПВЛ».

## Корневая причина

Не роутинг, а **клиентский стейт**. Два механизма сложились:

1. **Stale монотонный счётчик-«интент».** В `UserApp` есть `libraryOpenRequest` ([views/UserApp.jsx](../views/UserApp.jsx)): `useState(0)`, **только инкрементится** в `openPvlCourse` (кнопки «Учительская»/«ПВЛ»), **нигде не сбрасывается**. Garden-пункт «Библиотека» идёт через `handleViewChange('library')`, который его не трогал.

2. **Level-проверка в mount-эффекте + ремаунт компонента.** `CourseLibraryView` рендерится условно (`{view === 'library' && <CourseLibraryView/>}`) → при уходе из раздела **размонтируется**, при возврате — **монтируется заново**. Внутри — эффект:
   ```js
   useEffect(() => {
       if (!openPvlRequest || !canSeePvlCourse) return; // LEVEL-проверка, не EDGE
       setSelectedCourseId(PVL_ENTRY_COURSE_ID);        // → embedded ПВЛ
   }, [openPvlRequest, canSeePvlCourse]);
   ```
   React выполняет эффект **на каждом свежем маунте**. Если к этому моменту `openPvlRequest` truthy (остался ≥1 от прошлого входа в ПВЛ), `if (!openPvlRequest …)` не делает ранний выход → выбирается ПВЛ-курс → `courseSidebar.enabled=true` → меню переключается на PVL-набор.

Связка: после первого входа в ПВЛ счётчик «залип» на 1; каждый последующий Garden-«Библиотека» ремонтирует `CourseLibraryView`, и его mount-эффект читает stale truthy-значение **как свежий запрос открыть ПВЛ**. Меню переключается через `courseSidebar.enabled`, который true только при `selectedCourse === PVL_ENTRY_COURSE_ID`.

(`aiCampSession` из `localStorage` сам ПВЛ **не** открывает — единственный путь к `setSelectedCourseId(PVL_ENTRY)` это тот эффект. Поэтому чинить надо счётчик-интент, а не сессию.)

## Почему так получилось

1. **Триггер-счётчик прочитан как уровень.** Монотонный `n+1`-счётчик задумывался как «edge» (сработать на инкремент-клик), но в эффекте проверяется как «level» (`if (!openPvlRequest)`). Пока компонент жив, edge ≈ level (срабатывает на смену зависимости). Но при **unmount/remount** mount-прогон эффекта превращает любой ненулевой level в «событие» — задумка edge-семантики ломается.
2. **Счётчик, который только растёт, не может выразить «теперь хочу СПИСОК».** После того как его бампнули, нет способа вернуть «интент = не-ПВЛ», кроме явного сброса — которого не было.
3. **Диагностический ключ был внешним, не в коде.** Симптом «перебрасывает» выглядит как редирект. Но **URL не менялся** (зафиксировано через Claude in Chrome) → это не HTTP-редирект и не смена роута → искать надо в **клиентском стейте**, а не в маршрутах. Без этого наблюдения легко уйти копать роутер/guard'ы и не найти.
4. **Невидимый ремаунт.** `{cond && <View/>}` тихо размонтирует/монтирует дочерний компонент; его эффекты и локальный стейт сбрасываются. Mount-эффекты, читающие props-«интенты» из родителя, который пережил размонтаж, — классическая ловушка.

## Как починили

Минимум — 2 правки в `views/UserApp.jsx`, ничего в `CourseLibraryView`. Идея: на момент маунта `libraryOpenRequest` должен означать «нужен ли ПВЛ» (truthy только когда явно просили ПВЛ).

1. **Сброс интента на обычном заходе** — в `handleViewChange`, ветка `library`:
   ```js
   if (newView === 'library') {
       setLibraryResetToken((v) => v + 1);
       setLibraryOpenRequest(0); // обычный заход = список курсов, не реанимация ПВЛ
   }
   ```
2. **Ре-бамп ПОСЛЕ сброса** — в `openPvlCourse` порядок инвертирован:
   ```js
   const openPvlCourse = () => {
       handleViewChange('library');          // ставит libraryOpenRequest = 0
       setLibraryOpenRequest((n) => n + 1);  // 0 → 1 (функц. апдейт читает пост-значение)
       setMobileMenuOpen(false);
   };
   ```

Композиция функциональных апдейтов в одном батче даёт: Garden-«Библиотека» → `0` (список), «Учительская»/«ПВЛ» → `1` (ПВЛ). Детерминированно на каждое действие, счётчик больше не залипает. Оба набора меню целы. Подтверждено 4/4 на localhost под ментором + смоук на проде (deploy run `26877429633`).

## Что проверить в будущем

- **Симптом «перебрасывает», но URL не меняется → это client-state, не routing.** Не копать роутер/HTTP-редиректы; искать стейт-машину навигации (флаги вида `*Mode`, `*Request`, `selected*`), которая решает, что рендерить.
- **Триггер-счётчик/флаг, читаемый в `useEffect`, который монтируется заново.** Если компонент рендерится условно (`{cond && <X/>}`), его mount-эффекты прогонятся при каждом возврате. Любой «request»-проп из пережившего размонтаж родителя нужно либо **сбрасывать после использования**, либо сравнивать с предыдущим значением **на уровне родителя** (не в ремаунтящемся ребёнке — там ref сбросится). Edge-семантику нельзя строить на «truthy при маунте».
- **Монотонный счётчик как интент — антипаттерн, если интент бывает «выключен».** Либо сбрасывать его (как здесь), либо использовать явный булев/enum-интент, который можно поставить в любое значение.
- **`{view === X && <View/>}` тихо размонтирует.** При ревью навигации помнить: смена `view` = unmount/mount дочернего экрана, его локальный стейт и эффекты стартуют с нуля.
- **Сигнатура класса бага:** «работает с первого раза, ломается после первого использования смежной фичи» → почти всегда залипший/непрочищенный стейт-флаг.
