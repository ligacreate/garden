## Две популяции Лига-плательщиков
1. **On-platform (есть аккаунт, платит через кабинет):** логинится, `checkout` → возврат на `urlSuccess=?paid=1`.
2. **Off-platform (нет аккаунта / recurring через TargetHunter):** платит напрямую в Prodamus, нашу платформу не видит.

## Q1 — Prodamus «Платный контент»: per-товар, и с нашими динамическими товарами НЕ работает
- Поле **«Платный контент» — ПЕР-ТОВАР** (заполняется в **карточке товара** в кабинете Prodamus). До 4096 симв,
  можно ссылку на закрытый чат. Доставка: **email после оплаты** (нужно сделать поле Email обязательным) + на success-странице.
  ([help.prodamus.ru/…/platnyi-kontent](https://help.prodamus.ru/payform/priyom-oplaty/platnyi-kontent))
- **Наш checkout строит товары НА ЛЕТУ** (`products[0][name]=plan.title` инлайн в URL, [billingCheckout.mjs:119](../../push-server/billingCheckout.mjs)),
  это **не каталожные SKU** → поле «Платный контент» карточки товара к ним **не привязывается** → для нашего
  платформенного checkout email-доставка paid_content **не сработает**.
- Зато для **каталожных покупок (recurring/TargetHunter)** — сработает: если у Лига-товара в каталоге заполнить
  «Платный контент» = join-ссылки, каждый Лига-платёж через TH авто-отправит их на email. **Это ровно off-platform популяция.**

## Q2 — через наш URL/success-страницу
- Мы контролируем `urlSuccess`/`urlReturn` = `?paid=1` ([billingCheckout.mjs:123-126](../../push-server/billingCheckout.mjs)).
- `?paid=1` приземляет на **«Моя подписка»** ([ProfileView.jsx:614](../../views/ProfileView.jsx)), где уже есть `subActive = paid_until>=now`.
  → для **залогиненного** показываем join-кнопки прямо в карточке при `subActive`. (Это и есть прерванный фронт-запрос — маленький diff.)
- **Не-залогиненному** на `?paid=1` показывать ссылки **нельзя** — сессии нет, Лига-статус не верифицировать →
  показать «войдите в аккаунт, чтобы получить доступ». Реальная доставка для них — Prodamus-email (Q1), не наша страница.

## Q3 — не отдать не-Лига покупателю
- **Per-product paid_content** — по природе Лига-only: заполнено ТОЛЬКО на Лига-товаре; книга — другой товар, поля нет.
- **Платформенная карточка** гейтит на `paid_until>=now`, а после корневого фикса `paid_until` выдаётся **только за Лига-товар**
  → `paid_until>=now ⟺ активный Лига-подписчик`. Книгу-купивший `paid_until` не получает → кнопок не видит.

## Q4 — предлагаемый механизм (гарантия для Лига, ноль для книги)
**Два взаимодополняющих канала, оба Лига-гейтед:**
| Популяция | Канал | Гейт |
|---|---|---|
| On-platform (логин) | Кнопки «Вступить в канал/чат» в «Моя подписка» | `subActive` (paid_until>=now) |
| Off-platform / без аккаунта | «Платный контент» = join-ссылки на **Лига-каталог-товаре** в кабинете Prodamus (email) | per-product (только Лига-товар) |

**Опционально (унификация), чтобы и платформенный checkout слал email:** ссылаться в checkout на каталожный
Лига-товар (id) вместо ad-hoc `products[0][name]`, ЛИБО добавить свою SMTP-рассылку join-ссылок после
успешного `plan_payment`, гейтя по `order.plan_code ∈ Лига` (у нас plan_code уже есть, товаро-безопасно). Belt-and-suspenders.

## ⚠️ Downstream-нюанс (важно, связан с ботом)
Доставка ссылки ≠ авто-впуск. Off-platform плательщик **без профиля**, кликнув standing-ссылку → подаёт заявку →
поллер его **не одобрит** (hard-rule «незнакомца без профиля не трогаем») → заявка зависнет pending. Для гарантированного
впуска таких — либо завести им профиль (тогда поллер по email/username сматчит и одобрит), либо ручное одобрение Олей.
Т.е. для off-platform-без-аккаунта нужен ещё шаг «профиль», иначе ссылка есть, а бот не пускает.
