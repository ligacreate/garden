# Handover — авто-пауза подписки (FEAT-015), путь через BotHunter

**Дата:** 2026-06-11. **Статус:** разведка завершена, доделать сборку завтра.

## Проблема
FEAT-015 (авто-пауза при окончании подписки) **фактически не работает**. Вебхуки Prodamus/TargetHunter, доходящие до push-server, **безличны** — нет email, нет события «окончание подписки» (в `billing_webhook_logs` только `payment_success` без email/profile-match). Ни один из 52 активных профилей не привязан к Prodamus (`prodamus_subscription_id` NULL у всех). Поэтому платформа не паузит истёкших.
Пример: **Ольга Пограницкая** (`olyala2006@gmail.com`) — подписка кончилась ~10.06, в БД по-прежнему `status=active`.

## Решение (найдено)
Не через Prodamus-вебхуки, а через **бот BotHunter `ligaskrebeyko`**, который УЖЕ обрабатывает окончание подписки.
- Кабинет `bot.targethunter.ru`, бот «ligaskrebeyko» (id группы `67029a185ed01a40082ee64c`).
- **Цепочка «подписка закончилась»** (Автоматизация): триггер «запущена внешним источником» → ждёт ~25ч → исключает из 2 чатов Лиги (`-1002377682177`, `-1002432957741`) → переносит в список «Подписка закончена» → шлёт сообщение. Есть **зеркальная** цепочка возобновления (кнопка «Возобновить подписку»).
- В цепочке доступен контекст пользователя (работает `{Имя}`) → `@username`/ссылка на профиль доступны как переменные.
- Механизм отправки нам: блок действия **«Запрос во вне»** (POST JSON на наш URL; подтверждено их поддержкой + докой bot.targethunter.help/dev/api).

## Ключ матча — Telegram username (НЕ email)
- BotHunter **email не хранит** (`Email: -` у пользователей). Но хранит `@username` + TG-id.
- У **всех 14 активных ведущих** заполнен `profiles.telegram = https://t.me/<username>`. Пример: Пограницкая = `t.me/olgapogranitskaya` = `@olgapogranitskaya`.
- **Матч: `@username` (BotHunter) ↔ username из `profiles.telegram`, без учёта регистра.**

## Что доделать завтра
1. **Приёмник на платформе** (codeexec): endpoint, принимает `{username, event: expired|active}` → находит профиль WHERE `lower(username из telegram-ссылки) = lower(received)` → `status='suspended'` (expired) / `'active'` (active). Расширить push-server (там есть `applyAccessState`) или новый минимальный endpoint. Защита: проверять, что роль leader/mentor; идемпотентность.
2. **Конфиг блока «Запрос во вне»** (стратег даёт точный URL+тело, Ольга/настройщик вставляет): POST body `{username: <переменная @username/user_link>, event: 'expired'}` в цепочку окончания (после шага «Удаление пользователя»); аналогично `event:'active'` в цепочку возобновления.
3. **Тест** на одном (Пограницкая): прогнать цепочку → платформа ставит паузу.

⚠️ Правки бота — на **ЖИВОМ** боте (управляет доступом в канал). Только точечно, через настройщика, с тестом. Claude in Chrome для бота — строго read-only.
