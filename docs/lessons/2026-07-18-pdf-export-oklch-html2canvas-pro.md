# PDF/PNG-экспорт падал на oklch — html2canvas 1.4.1 не парсит цвета Tailwind v4

**Дата:** 2026-07-18
**Связано:** BACKLOG `BUG-PDF-EXPORT-OKLAB-FAIL` (P2, заведён 2026-05-11), решение — вариант A.

## Симптом
В Конструкторе при экспорте PDF (воркбук/сценарий/материал) пользователь видел alert «Ошибка при создании PDF», файл не скачивался. В консоли — `Attempting to parse an unsupported color function "oklch"`. Ноль успешных PDF за месяцы. Тот же дефект молча ломал скачивание PNG-карточки отзыва на странице ведущей.

## Корневая причина
Стек на **Tailwind v4** (`@import "tailwindcss"`), который в проде отдаёт цвета через `oklch()`/`oklab()`. Растеризатор **`html2canvas@1.4.1`** не умеет парсить эти color-функции и бросает при обходе computed styles экспортируемого узла. Ошибка ловилась в `try/catch` внутри `handleExportPdf` → пользователь видел alert, а мониторинг (MON-001) молчал, т.к. это не uncaught exception.

Слой источника правды — **зависимость-растеризатор**, а не код экспорта. Код экспорта корректен; дырявой оказалась библиотека, не поспевшая за цветовой моделью CSS.

## Как починили
Вариант **A** — замена растеризатора на форк, понимающий современные color-функции:
- `npm i html2canvas-pro` (`^2.2.4`), `npm uninstall html2canvas`.
- Своп динамического импорта в **обеих** реальных точках: `views/BuilderView.jsx:246` (PDF: воркбук/сценарий/материал — все через один `handleExportPdf`) и `views/LeaderPageView.jsx:316` (PNG-карточка отзыва). API идентичен (`default`-экспорт = та же функция, те же опции), jsPDF не трогали.
- Старый `html2canvas` остаётся **транзитивно** (optional-dep `jspdf`, а также неиспользуемая прямая зависимость `html2pdf.js`) — в наших путях не грузится.

## Что проверить в будущем (сигналы похожих багов)
- **Любой `html2canvas`/canvas-растеризатор на стеке Tailwind v4** — подозревай oklch/oklab. При апгрейде Tailwind до версии с новой цветовой моделью проверяй ВСЕ пути растеризации.
- **Ищи все точки импорта, а не только названную.** Баг был в задаче про BuilderView, но идентичный паттерн `import('html2canvas')` жил в LeaderPageView — параллельный баг того же типа. `grep import('html2canvas')` по всему `views/`.
- **Caught-ошибка ≠ наблюдаемая.** Если фича молча не работает у всех, а мониторинг тих — ищи проглоченный `catch` (здесь нужен был бы `reportClientError` в catch'е, чтобы увидеть раньше мая).
- **Верификация — на корне бага.** Доказательство фикса: реальный oklch-DOM в реальном браузере, старый lib бросает — новый рендерит. Это надёжнее, чем гонять весь auth-gated flow.
