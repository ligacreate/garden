## Что собрать

### 1. Schema `shop_items` в БД

Через ssh+psql под gen_user:

```sql
\d public.shop_items
SELECT * FROM shop_items LIMIT 5;
SELECT count(*) FROM shop_items;
```

Особенно интересует:
- Какие колонки есть сейчас (title, image, price, old_price,
  description, promo_code, contact_method, kind/type, etc.)
- Есть ли поля под "тип товара" / "способ покупки"
- Есть ли поле для размеров (XS/S/M/L/XL — на скриншоте у футболки)
- Есть ли поле под цифровой товар (download_url, file_url)

### 2. Текущая админка `ShopAdmin`

`views/AdminPanel.jsx`, где-то на строке 1591 (по recon
`_02` от 2026-05-07). Прочитать полностью компонент:
- Что уже редактируется в форме (какие поля)
- Какие endpoint'ы используются
- Есть ли upload изображений
- Есть ли превью

### 3. Витрина магазина для ведущих

Найти view, который рендерит магазин в обычном UserApp (не
AdminPanel). Скорее всего:
- `views/ShopView.jsx` или похожее
- Может быть inline в каком-то view

Прочитать:
- Какой компонент рендерит карточки
- Откуда берётся подзаголовок «Товары напрямую от производителя»
  (нужно убрать)
- Логика выбора кнопки «Связаться» vs «Перейти» (по какому полю
  определяется?)
- Размеры (XS/S/M/L/XL) — где живут? hardcoded? отдельная таблица?
  поле в shop_items?
- Старая цена / скидка — отдельные поля или вычисляется?

### 4. Service-layer

В `services/dataService.js` (или другом) — методы для shop_items:
- get / list / create / update / delete
- Какие endpoint'ы PostgREST

### 5. RLS на `shop_items`

```sql
SELECT polname, polcmd, pg_get_expr(polqual, polrelid) AS using_expr
FROM pg_policy
WHERE polrelid = 'public.shop_items'::regclass;
```

Кто видит, кто редактирует.
