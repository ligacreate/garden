## Где лежат данные feedback'а

### A. Ответы студентки

`pvl_student_homework_submissions.payload` (jsonb):
```
{
  "thread": [...],
  "versions": [
    {
      "isCurrent": true/false,
      "createdAt": "...",
      "answersJson": {
        "qb-qa-1": "02.05.26",
        "qb-0c597aa3": "<b><p><strong>HTML-ответ</strong></p></b>",
        ...
      },
      "textContent": "",
      "versionNumber": 1
    }
  ]
}
```

- Key `qb-<id>` — abstract вопрос, mapping в frontend-конфиге
  (НЕ в БД). Без mapping'а — голый id, нечитаемо.
- Value — HTML rich-text (p, strong, b, ul, li, br).
- Versions может быть несколько (студентка дорабатывала).
  `isCurrent: true` — финальная.

### B. Комментарии ментора

`pvl_homework_status_history.comment` (text):
- HTML rich-text (длинные, multi-paragraph).
- Связан с submission через `submission_id`.
- `changed_by` — UUID ментора (можно через JOIN profiles
  получить имя ментора).
- `from_status`/`to_status` — переход (например,
  draft→in_review «Отправлено на проверку», in_review→accepted
  «Оля, спасибо за задание...»).

**Sample реального комментария ментора** (Василина Лузина,
2026-05-09):
> «Лилия, добрый день. По заданию, что нужно доработать:
> 1. Сформулировать результат письменной практики...
> 2. Доработать инструкцию, разбить её на пункты...
> 3. В рефлексию добавить один вопрос...» (~1000 символов)

Это и есть feedback, который Ольге нужен.

## Архитектурный план (предварительный)

1. **Frontend-only решение** — никаких новых RPC.
   - На дашборде FEAT-017 кнопка «Выгрузить ДЗ» (рядом с фильтром
     когорты).
   - При клике: `pvlPostgrestApi.listStudentHomeworkSubmissions`
     для каждой студентки в когорте + JOIN на homework_items +
     JOIN на status_history (отдельный fetch) + JOIN на profiles
     (для имён менторов).
   - Сборка CSV на клиенте.
2. **Альтернатива** — серверный RPC `pvl_admin_export_homework(p_cohort_id)`
   который собирает всё одним JOIN'ом и возвращает jsonb-массив
   готовых строк. Проще на фронте, чище для больших объёмов
   (когда будет 100+ студенток).

Для текущих масштабов **Frontend-only достаточно**. Если
Ольга захочет — RPC сделаем отдельно.
