## 1. Инцидент «Load failed» при входе Рухшаны
**Симптом:** на liga.skrebeyko.ru при входе — «Load failed», кнопка виснет «Входим…».

**Что проверили (всё зелёное):** auth-сервис (health 200, login 401/JSON за 0.7с), CORS (preflight 204, ACAO=liga), api/push/liga (200), service worker (минимальный, вход не рубит), аккаунт (есть, `has_pw=t`), IPv6 (AAAA нет — не тот случай), fail2ban/rate-limit/firewall (нет).

**Корень (по access-логу Caddy):** в окно её попытки **не было ни одного `POST /auth/login` с её устройства** — запрос **не дошёл до сервера**. Одновременно другие клиенты логинились успешно. → Проблема на участке её устройство→сеть→сервер (разовый сетевой сбой / Private Relay / DNS), **не платформа**.

**Разрешение:** Рухшана повторила с другого момента, старый пароль не подошёл (честный 401) → сбросила через «Забыли пароль?» → вошла (200). Всё видно в логе: `login 401 → request-reset ×2 → reset 200 → login 200 → /auth/me 200`.

**Важный побочный вывод:** ранний «G1 SMTP-тест» на skrebeyko@proton.me дал 200, но письмо **не ушло** — этот email не в `users_auth`, `request-reset` отдаёт silent-ok (анти-энумерация). Аккаунт Оли на платформе — **olga@skrebeyko.com**. Повторный тест туда — письмо дошло, SMTP подтверждён.

## 3. Продуктовое решение — жёсткий замок Лиги
Вопрос участницы «а если выкинуло, смогу зайти?» → проверили и **заметили**: по текущему дизайну «В1 / кабинет-первый» **неоплата НЕ закрывает платформу.** Ночной reconcile Лига-роли лишь помечает `subscription_status='overdue'`, а `access_status` оставляет `active` → RLS пропускает → платформа видна. Режется только Лига-специфика (`subActive`/paid_until): кнопки входа в TG-сообщество.

**Оля: это неправильно** — платформа тоже привилегия Лиги, неоплата должна закрывать доступ.

**Превью (read-only):** жёсткий замок закроет **ровно 6 истёкших стажёров** (Исламова, Баженова, Шайтанова, Громова, Ильиных, Ивашова — не платят 5–40 дней). Бартер/годовики (Кокорина, Дегожская — exempt) защищены. Оплаченные 33 — не тронуты. Переблокировки нет.

**Решение (diff-on-review 268):** скоуп leader/mentor/intern, grace 3 дня (единый с TG-киком), без session-bump (не нужен — RLS + 60-сек авто-логаут фронта уже разлогинивают).

## 4. Что сделали (технически)
1. **Фикс join-poller (267) — ЗАДЕПЛОЕНО.** Корень флака `fetch failed`: `tgGet` на голом `fetch` без IPv4/таймаута; IPv6 к api.telegram.org = ENETUNREACH, happy-eyeballs Node 20 иногда шёл по v6.
