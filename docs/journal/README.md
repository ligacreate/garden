# docs/journal/

**Журнал сессий, инцидентов и одноразовых решений. Хронология, а не справочник.**

Сюда складываем датированные артефакты:

- `EXEC_YYYY-MM-DD_*` — лог исполнения миграций/фаз.
- `HANDOVER_YYYY-MM-DD_*` — передача между сессиями/командами.
- `REPORT_YYYY-MM-DD_*` — разовые отчёты и аудиты.
- `RECON_YYYY-MM-DD_*` — разведка перед задачей.
- `DECISION_YYYY-MM-DD_*` — зафиксированное решение по развилке.
- `INCIDENT_YYYY-MM-DD_*` — пост-мортем инцидента.
- `MIGRATION_YYYY-MM-DD_*` — описание разовой миграции.

Эти файлы фиксируют контекст момента — не считать их актуальной
документацией. Если из инцидента родилось общее правило — оно идёт в
[../lessons/](../lessons/), не сюда. Если из решения родилась живая
функция — она идёт в [../../plans/](../../plans/).
