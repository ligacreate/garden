# Урок: CSS `capitalize` ломает русское «г.» в заголовке календаря

**Дата:** 2026-06-19. **Где:** components/CalendarWidget.jsx, views/MeetingsView.jsx.

## Симптом
Заголовок календаря показывал «Май 2026 **Г.**» — единица года «г.» выводилась с заглавной буквы.

## Корневая причина
CSS `text-transform: capitalize` (Tailwind-класс `capitalize`) капитализирует первую букву **каждого слова**, а не строки. Для строки `«май 2026 г.»` это даёт `«Май 2026 Г.»`. Слой-владелец — формат данных (sentence-case), а не presentation: правильно капитализировать первую букву строки в JS, а не CSS-трансформом по словам.

## Почему так получилось
`capitalize` читается как «сделать с заглавной», но семантика — per-word. Для русских строк с аббревиатурами/единицами в конце (`г.`, `шт.`, `руб.`, `мск`) это всегда баг. Эталонный sentence-case уже был в коде (`PvlCalendarBlock.jsx:119`, normalize-хелпер в PracticesView), но MeetingsView/CalendarWidget разошлись с ним и применяли CSS `capitalize`.

## Как починили
Убрали CSS `capitalize`; капитализируем только первую букву строки в JS:
`const s = date.toLocaleString('ru-RU',{month:'long',year:'numeric'}); s.charAt(0).toUpperCase()+s.slice(1)`.
Плюс `whitespace-nowrap` на заголовок (месяц между стрелками не должен переноситься). Дни недели (ПН ВТ…) — норма календаря, не трогали.

## Что проверить в будущем
- `grep -rn "capitalize" views/ components/` — каждый случай, где в строке есть слово-аббревиатура/единица, — кандидат на этот баг.
- Для отображения русских дат/единиц предпочитать sentence-case в данных (JS), а не `text-transform: capitalize`.
- При появлении нового календаря/датового заголовка — переиспользовать существующий sentence-case паттерн, не вводить `capitalize`.
