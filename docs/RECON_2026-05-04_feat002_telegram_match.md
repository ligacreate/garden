## Алгоритм матчинга (краткое описание)

`scripts/feat002-tg-recon/match_to_garden.py`:

1. **Нормализация**: lowercase, удаление пунктуации/эмодзи, разбиение
   по пробелам. Токены ≥ 2 симв.
2. **Подмножество** (high): `g_tokens ⊆ t_tokens` или наоборот, при
   условии что меньший сет содержит ≥ 2 токенов.
   - Защита: одиночные токены («Светлана», «Лена») НЕ дают high — это
     ловило Колотилову на «Светлана» и Лену Ф на «Лена».
3. **Пересечение токенов** (medium): score зависит от длины самого
   длинного общего токена (фамилия > имя для tie-break).
4. **Транслит ru→latin** для случаев когда garden — кириллица, TG —
   латиница. (Не покрывает украинскую кириллицу — отсюда промах с
   Лилія Малонґ.)
5. **Левенштейн ≤ 2** на токенах ≥ 4 симв (опечатки/транслит).
6. **Substring match** (medium слабый, score 0.4) — для псевдонимов
   типа «Настин фиксик» ↔ «Анастасия Фиксикова».
7. **1-к-1 distinctness**: жадное назначение по убыванию score, любой
   принятый матч блокирует TG-юзера для остальных кандидатов.

Тип сигнала записан в `match_result.json` (поле `why`) для каждой
строки, можно перепроверить.
